# ATO v0.2.0 Productization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship ATO v0.2.0 as a clean-installable PyPI plus npm product with truthful parallel execution, durable approvals, isolated tasks, strict CI, and verified public release artifacts.

**Architecture:** `ato_core` owns orchestration, state, tools, approvals, persistence, and bridge schemas. TypeScript packages are protocol and presentation adapters that invoke `python -m ato_core.bridge`; every task stores state under its own directory and LangGraph parallel branches return reducer-safe deltas.

**Tech Stack:** Python 3.10+, Pydantic 2, LangGraph, SQLite checkpointing, Hatchling, pytest, TypeScript, Node.js 18+, MCP SDK, Commander, pnpm 9, Vitest, GitHub Actions, PyPI, npm.
